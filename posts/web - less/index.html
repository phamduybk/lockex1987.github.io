CSS3

CSS seems to be a programming language that was often underestimated. But it's different now. Its latest version 3 has many improvements. Community also contribute with frameworks (Bootstrap) and tools (LESS).

Compile less file to css file using lessc (Node.js).

lessc file.less file.css

Let's start with LESS.

1. Cleaner Structure with Nesting

In CSS, we write out every rule set separately, which often leads to long selectors that repeat the same stuff over and over. Here's a typical example:

#header {}
#header #nav {}
#header #nav ul {}
#header #nav ul li {}
#header #nav ul li a {}

LESS allow to nest rule sets inside other rule sets, as a way to show hierarchy. Let's rewrite above example with nesting:

# header {
    #nav {
        ul {
            li {
                a {}
            }
        }
    }
}

Also, if you want to give pseudo-classes this nesting structure, you can do so with the & symbol. Pseudo-classes are things such as :hover, :active and :visited. Your code would look as follows:

a {
    &:hover {}
    &:active {}
    &:visited {}
}

2. Variables for Faster Maintenance

LESS allows us to specify variables in one place - such as for brand colors, border lengths, side margins and so on - and then reuse the variables elsewhere in the style sheet. The value of the variable remains in one place, so making a change is as simple as changing that one line.

Variables start with an @ and are written like this:

@brandColor: #4455EE;

#header { background-color: @brandColor; }
#footer { color: @brandColor; }
h3 { color: @brandColor; }

3. Reusing Whole Classes

Variables are great, but we often reuse more than single values.A good example is code that's different for every browser, like the CSS3 property border-radius. We have to write at least three declarations just to specify it:

#login-box {
    -webkit-border-radius: 5px;
    -moz-border-radius: 5px;
    border-radius: 5px;
}

LESS solve this by allowing us to reuse whole classes simply by referencing them in our rule sets. For example, let's create a new classes for the above border-radius and reuse it in another rule set:

.round-corner {
    -webkit-border-radius: 5px;
    -moz-border-radius: 5px;
    border-radius: 5px;
}
#login-box {
    .round-corner;
}

Now #login-box will inherit the properties of rounded-corner class. But what if we want more control over the size of the corners? No problem. We can pass along variables to the "mixin" (these reusable classes are called mixins) to get a more specific outcome.

.round-corner(@radius: 5px) {
    -webkit-border-radius: @radius;
    -moz-border-radius: @radius;
    border-radius: @radius;
}
#login-box {
    .round-corner(3px);
}

4. Operations

Variables let us specify things such as common palettes of colors, but what about relative design elements, like text that's a bit lighter than the background, or the inner border that one pixel thicker than the outer border?

@base-margin: 25px;
#header { margin-top: @base-margin + 10px; }
