Save git password

When I push sources from local to GitHub, it keep asking me for the username and password. It is very tedious.

This tutorial will guide you to skip typing git password.

Fortunately, from version 1.7.9 (released in late January 2012), Git has a credentials system that can help with this. Git has a few options provided in the box:

  • The default is not to cache at all. Every connection will prompt you for your username and password.
  • The “cache” mode keeps credentials in memory for a certain period of time. None of the passwords are ever stored on disk, and they are purged from the cache after 15 minutes.
  • The “store” mode saves the credentials to a plain-text file on disk, and they never expire. This means that until you change your password for the Git host, you won’t ever have to type in your credentials again.

Find a helper: 

git help -a | grep credential

Read its description:

git help credential-store

Tell git to use it:

git config --global credential.helper store

Cache mode

Command:

git config --global credential.helper cache

Git will save your password in memory for some time. By default, Git will cache your password for 15 minutes.

Set the cache to timeout after 1 hour (setting is in seconds):

git config --global credential.helper 'cache --timeout=3600'

Store mode

Command:

git config --global credential.helper store
The point of this helper is to reduce the number of times you must type your username or password. For example: 
$ git config --global credential.helper store
$ git push http://example.com/repo.git
Username: <type your username>
Password: <type your password>

[several days later]

$ git push http://example.com/repo.git
[your credentials are used automatically]

Your passwords are stored in a plain file in your home directory. The default is ~/.git-credentials.

The content of the file:

https://lockex1987:PLAIN_PASSWORD@github.com

IMPORTANT: The password is not encrypted. It's just encoded by HTML URL Encoding, and can be decoded easily. Therefore, keep the file securely.
