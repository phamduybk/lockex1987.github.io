7. Giao diện chương trình

Cấu trúc layout của các trang đều được dựa trên trang baseLayout.jsp hoặc popupLayout.jsp. Các trang chỉ khác nhau ở phần nội dung, còn bố cục đều giống nhau. Các trang này ở thư mục /jsp/layout/. Thư mục này còn chứa các file khác: externalCssJs.jsp (nơi khai báo include các file JS, CSS), processing.jsp (hiển thị thanh processing).

Framework sử dụng các file CSS sau:
  • jquery-ui-1.8.13.custom.css (CSS của JQuery)
  • validationEngine.jquery.css (CSS của Validation engine)
  • notify-osd.css (CSS của cơ chế phản hồi)
  • ui.dynatree.css (CSS của cây)
  • displaytagex.css (CSS của display table)
  • layout.css (CSS của bố cục chương trình)
  • menu.css (CSS dùng riêng cho menu)
  • theme.css (CSS hay dùng trong framework)
Một giao diện đẹp hay không còn tùy thuộc vào thẩm mỹ của từng người, tùy thuộc vào mục đích chương trình (chương trình giải trí cần bắt mắt, chương trình quản lý cần rõ ràng, dễ sử dụng,...). Chúng ta nên phát triển thành các chủ đề khác nhau (bố trí các file CSS vào các thư mục chủ đề riêng). Các dự án có thể tùy chọn chủ đề phù hợp.

Trong quá trình phát triển chương trình, dev hạn chế tối đa việc thiết lập giao diện qua thuộc tính style của các thẻ. Dev cần sử dụng các class sau:
Thẻ Mục đích
borderDiv Tạo border đơn giản
fieldset, legend Tạo vùng bao cho nội dung
label Để hiện thị nhãn của các form
multiCheckbox Dùng khi sử dụng danh sách checkbox
viewContent Dùng để hiện thị nội dung ở trang xem chi tiết
ticker Dùng để hiển thị đối tượng đang được thao tác
activeButton, inactiveButton, roundContainer Dùng để tạo nút đơn giản
myTree Dùng cho tree, nếu cần viết lại tree
dataPicker Dùng cho dataPicker, nếu cần phải viết lại dataPicker
Các class này được định nghĩa trong file theme.css.
