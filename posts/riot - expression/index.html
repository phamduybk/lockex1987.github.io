Biểu thức Riot

Biểu thức

Mã HTML có thể kết hợp với các biểu thức bên trong dấu ngoặc nhọn:

x

Các biểu thức có thể thiết lập cho thuộc tính hoặc node con.

x

Các biểu thức là 100% JavaScript. Các ví dụ là:

{ title || 'Untitled' }
{ results ? 'ready' : 'loading' }
{ new Date() }
{ message.length > 140 && 'Message is too long' }
{ Math.round(rating) }

Mục tiêu là để các biểu thức càng nhỏ, càng đơn giản càng tốt. Nếu biểu thức phức tạp, bạn nên để logic vào trong sự kiện update. Ví dụ:

<my-tag>

  <!-- the `val` is calculated below .. -->
  <p>{ val }</p>

  // ..on every update
  this.on('update', function() {
    this.val = some / complex * expression ^ here
  })
</my-tag>

Thuộc tính boolean

Các thuộc tính boolean (checked, selected,...) được bỏ qua nếu giá trị của biểu thức là falsy. Ví dụ <input checked={ null }> sẽ trở thành <input >.

W3C nói rằng các thuộc tính boolean là true nếu có thuộc tính - kể cả khi giá trị là rỗng.

Biểu thức sau không chạy được:

<input type="checkbox" { true ? 'checked' : '' }>

Lý do là chỉ các biểu thức thuộc tính hoặc là text là hợp lệ.

Class

Riot có một cú pháp đặc biệt cho các class CSS. Ví dụ:

<p class={ foo: true, bar: 0, baz: new Date(), zorro: 'a value' }> ... </p>

Đánh giá thành foo baz zorro.

Hiển thị dấu ngoặc nhọn

Bạn có thể hiển thị một biểu thức mà không đánh giá nó bằng cách escape dấu ngoặc mở:

\{ this is not evaluated }

sẽ output ra:

{ this is not evaluated }

Hãy chắc chắn escape các dấu ngoặc ở bất cứ trường hợp nào mà nó không nên được Riot đánh giá. Ví dụ như biểu thức chính quy sau sẽ không validate được input phải là 2 ký tự số mà sẽ trở thành một ký tự số theo sau bởi số "2":

<input type="text" pattern="\d{2}">

Cú pháp đúng sẽ là:

<input type="text" pattern="\d\{2}">

Tùy chỉnh dấu ngoặc

Bạn có thể tùy chỉnh dấu ngoặc nếu muốn, ví dụ:

riot.settings.brackets = '${ }'
riot.settings.brackets = '\{\{ }}'

Phần mở và phần đóng được cách nhau bởi một ký tự space.

Các biểu thức ở trong thẻ style sẽ được bỏ qua.
