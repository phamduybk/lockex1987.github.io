XML

Dù JSON đã là rất tốt rồi, nhưng chúng ta vẫn có lúc phải dùng XML (các dự án cũ).

Chúng ta sẽ thử parse, tạo file XML với Java.

Trong JDK, có 3 parser sẵn có - DOM, SAX, JAXB. Mỗi cái thì đều có ưu và nhược điểm.

DOM

DOM là cách sử dụng dễ nhất. Nó parse toàn bộ văn bản XML và tải vào trong bộ nhớ với cấu trúc cây. DOM parser thì chậm và tốn bộ nhớ nếu nó phải tải văn bản XML với nhiều dữ liệu.

Code cũng dài.

SAX

SAX (Simple API for XML) làm việc không giống như DOM. Nó không tải văn bản XML gì vào trong bộ nhớ và tạo đối tượng thể hiện văn bản XML. Thay vào đó, SAX sử dụng các hàm callback để thông báo cấu trúc của XML.

Các hàm callback là:

  • startDocument, endDocument
  • startElement, endElement
  • characters

SAX thì nhanh và tốn ít bộ nhớ hơn DOM.

Code khó.

JAXB

JAXB (Java Architecture for XML Binding) cũng có sẵn trong JDK (từ bản 6.0), sử dụng annotation để convert giữa đối tượng Java và XML.

  • marshalling: Đối tượng Java --> XML
  • unmarshalling: XML --> Đối tượng Java

Code khá dễ:

  • Thêm annotation (@XmlRootElement, @XmlElement, @XmlAttribute)
  • Sử dụng jaxbMarshaller.marshal() hoặc jaxbUnmarshaller.unmarshal()
