Server OS: CentOS

Web server: nginx

Giả sử bạn deploy một ứng dụng có tên là spam-email. Hãy thay tên spam-email bằng tên tương ứng của bạn ở các câu lệnh và cấu hình.

Copy thư mục spam-email vào thư mục /var/www/html

Không cần copy thư mục .git, node_modules, resources, tests.

Phân quyền thư mục cache và storage:

sudo su
cd /var/www/html/
cd spam-email
chmod -R 777 bootstrap/cache
chmod -R 777 storage

Thêm file spam-email.conf vào thư mục /etc/nginx/conf.d/ với nội dung như sau:

server {
    listen       80;
    server_name spam-spam.dev.com;
    root /var/www/html/spam-email/public;
    index index.php index.html index.htm;
    access_log  /var/log/nginx/spam-email.access.log  main;
    error_log /var/log/nginx/spam-email.error.log error;
    
    location / {
        index  index.html index.htm index.php;
        try_files $uri $uri/ /index.php?$query_string;
    }

    error_page   404              /404.html;
    error_page   500 502 503 504  /50x.html;
    location = /50x.html {
        root   /usr/share/nginx/html;
    }

    location ~ \.php$ {
        include fastcgi_params;
        fastcgi_split_path_info ^(.+\.php)(/.+)$;
        fastcgi_pass   127.0.0.1:9000;
        fastcgi_index  index.php;
        fastcgi_param  SCRIPT_FILENAME  $document_root$fastcgi_script_name;
        include        fastcgi_params;
    }

    location ~ /\.ht {
        deny  all;
    }
}

Restart nginx

systemctl restart nginx
